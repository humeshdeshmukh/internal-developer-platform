Platform Engineering: Self-Service Cloud

Platform Engineering is the discipline of designing and building toolchains and workflows that enable self-service capabilities for software engineering organizations in the cloud era.

Aegis IDP Philosophy: The goal of Aegis Developer Portal is to reduce cognitive load on developers by providing golden paths (standard templates) instead of locking infrastructure behind manual service tickets.

Core Architectural Concepts

By abstracting cloud providers behind unified infrastructure templates, engineers can deploy full-stack applications with their corresponding datastores without needing to master complex Kubernetes manifests or Terraform providers.

  • Productivity: Zero wait time for sandbox and dev environment provisioning.
  • Security: Automatically inject security scanning checks (like Trivy, Falco) during builds.
  • Governance: Enforce resource quotas and labels dynamically at provision time.

Infrastructure & Security Standards

All resources generated by Aegis software templates adhere to pre-approved corporate security controls and resource boundaries.

Resource Limits (Quotas)

To ensure cluster health and budget predictability, namespaces are capped using resource quotas:

# Default Sandbox Quota limits cpu_hard_quota: 2 cores memory_hard_quota: 4Gi storage_hard_quota: 20Gi

Database Encryption

Databases created via the database template inject credentials as Kubernetes Secrets which are reference-mapped by stateful workloads, keeping clear-text values out of version control systems.

Developer Experience (DevEx) Best Practices

Developer Experience centers around how friction-free it is for a developer to write, test, build, and deploy production software.

The Golden Path

A golden path is a pre-packaged, fully supported method of building and deploying software. By using Aegis templates, you are automatically aligned with:

  • Continuous Integration (CI) unit tests.
  • Secure Docker base images (distroless or alpine).
  • ArgoCD GitOps synchronization loop (self-healing deployment reconcile).
